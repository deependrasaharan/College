================================================================================
ALGORITHMS IMPLEMENTED
================================================================================

1. HEURISTIC APPROACH: Min-Conflicts with Random Restarts
   ---------------------------------------------------------
   Method: Local search with stochastic moves
   
   Process:
   - Start with random configuration
   - Iteratively reduce conflicts by moving queens
   - 5% random walk to escape local minima
   - When solution found, restart with new random board
   - Continue until time limit
   
   Characteristics:
   - Finds diverse solutions (random restarts)
   - Variable performance (stochastic)
   - Good for large N
   - May take many iterations per solution
   
   Performance (N=8, 10s):
   - Found: 92 solutions
   - Time: 10.000s
   - Iterations: 875,309
   - Rate: 9.2 solutions/second
   - Efficiency: 9,514 iterations/solution

2. BACKTRACKING APPROACH: Systematic Search
   ------------------------------------------
   Method: Depth-first search with pruning
   
   Process:
   - Place queens row by row
   - For each row, try all columns
   - Check if placement is safe before continuing
   - When all rows filled, save solution
   - Backtrack and continue searching
   - Stop when time limit reached
   
   Characteristics:
   - Systematic exploration
   - Finds solutions in order
   - No duplicates generated
   - Very efficient for moderate N
   - Predictable performance
   
   Performance (N=8, 10s):
   - Found: 92 solutions (all solutions!)
   - Time: 0.005s (finished early!)
   - Nodes: 2,057
   - Rate: 18,771 solutions/second
   - Efficiency: 22.4 nodes/solution
